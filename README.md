# Content Radar MVP

Content Radar MVP — демо-инструмент для маркетолога, который сравнивает ваш канал с конкурентами и показывает, **что делать дальше**, чтобы расти по вовлечённости.

Проект собран как хакатонный MVP: простая архитектура, понятный UI, прозрачная аналитика и честная обработка ограничений источников.

---

## 1) User-case (главный сценарий)

### Задача пользователя
Маркетолог хочет быстро понять:
- кто из конкурентов лидирует по вовлечённости;
- какие темы, форматы и слоты публикации дают лучший отклик;
- какие конкретные шаги нужно сделать в своём контент-плане.

### Что делает пользователь
1. Открывает дашборд (`/`).
2. Выбирает режим:
   - загрузка данных из файла (CSV/JSON), или
   - сбор постов по публичной ссылке.
3. (Для режима ссылки) получает посты, выбирает нужные, сохраняет выбор.
4. Запускает анализ.
5. Получает:
   - рейтинг каналов;
   - блок победителя и разрыв от лидера;
   - топ тем/форматов/времени;
   - график динамики вовлечённости;
   - практические рекомендации «что делать в ближайшие 7–14 дней».

## 2) Архитектура

### Технологии
- **Backend:** Python 3.13, FastAPI, Pydantic
- **Frontend:** HTML + CSS + Vanilla JavaScript (Jinja2-шаблон)
- **Хранение:** in-memory (для MVP-скорости)

### Структура проекта

```text
.
├── app/
│   ├── main.py                  # API-слой и роуты
│   ├── models.py                # Pydantic-модели запросов/ответов
│   ├── services/
│   │   ├── analyzer.py          # расчёт метрик, топы, рекомендации
│   │   ├── discovery.py         # сбор по ссылке + нормализация + fallback
│   │   └── parsers.py           # парсинг/валидация CSV/JSON
│   ├── static/
│   │   ├── app.js               # клиентская логика
│   │   └── styles.css           # стили дашборда
│   └── templates/
│       └── index.html           # UI-шаблон
├── data/
│   └── demo_posts.json          # демо-набор постов
├── scripts/
│   └── smoke_scenarios.py       # smoke-проверки ключевых сценариев
├── requirements.txt
└── README.md
```

### Логические слои
1. **API-слой (`main.py`)**  
   Принимает запросы, валидирует вход, вызывает сервисы, возвращает JSON для UI.

2. **Слой моделей (`models.py`)**  
   Формализует контракты API (типы, диапазоны, обязательные поля).

3. **Сервисный слой (`services/*`)**  
   - `parsers.py` — безопасный и строгий разбор файлов;
   - `discovery.py` — сбор постов по публичным ссылкам + demo fallback;
   - `analyzer.py` — формула вовлечённости, агрегирование, инсайты.

4. **UI-слой (`templates + static`)**  
   Рендерит дашборд и управляет режимами/визуализацией аналитики.

## 3) Логика продукта

### 3.1 Источники данных
- **Файл** (`/upload_posts`): CSV/JSON с полями постов.
- **Ссылка** (`/discover_posts`): попытка получить открытые посты из публичной страницы.
- **Demo** (`/load_demo`): гарантированный рабочий набор для презентации MVP.

### 3.2 Поток обработки
1. Данные загружаются/собираются.
2. Все посты приводятся к единой модели `PostRecord`.
3. Для каждого поста считается `engagement_score`.
4. Строятся агрегаты:
   - средняя вовлечённость;
   - топ постов;
   - топ тем;
   - лучшие форматы;
   - лучшее время публикации.
5. Формируются рекомендации в формате действий для маркетолога.

### 3.3 Формула вовлечённости

```text
Engagement Score = likes × 1 + comments × 2 + shares × 3 + views × 0.1
```

Формула возвращается через API (`engagement_formula`) и показывается в UI для прозрачности аналитики.

### 3.4 Сравнение каналов
Эндпоинт `/compare_channels`:
- собирает данные по вашему каналу и конкурентам;
- строит рейтинг и определяет лидера;
- считает разрыв от лидера;
- возвращает инсайты и план действий для сокращения отставания.

## 4) Безопасность

В MVP уже реализованы базовые меры защиты:

1. **SSRF-защита в discovery-слое**
   - разрешены только `http/https`;
   - блокируются `localhost`, loopback, private/link-local/reserved IP;
   - выполняется проверка DNS-резолва на небезопасные адреса.

2. **Ограничения входных данных**
   - строгая валидация через Pydantic (`min/max`, обязательные поля, типы);
   - лимиты на размеры и формат файлов (`.json/.csv`, max 5MB).

3. **Fail-safe поведение источников**
   - при недоступности/ограничении публичных данных включается **demo fallback**;
   - пользователь получает явный warning, а не «тихую» ошибку.

4. **Безопасная обработка сетевого сбоя**
   - сетевые/парсинговые исключения не ломают пользовательский поток;
   - сервис возвращает контролируемый результат для demo-режима.

### Ограничения безопасности/данных (честно)
- Платформы типа Instagram/Telegram могут ограничивать публичные метрики без официального API.
- Текущее хранение — in-memory (данные не персистентны после рестарта).
- Для production-уровня стоит добавить БД, auth, rate limiting, audit-логирование и секрет-менеджмент.

## 5) API (ключевые эндпоинты)

- `GET /` — дашборд
- `GET /health` — health-check
- `POST /upload_posts` — загрузка CSV/JSON
- `POST /discover_posts` — сбор постов по ссылке
- `POST /select_posts` — сохранить выбранные посты
- `POST /analyze_content` — запустить анализ
- `POST /compare_channels` — сравнить ваш канал и конкурентов
- `GET /insights` — получить инсайты/рекомендации
- `POST /load_demo` — загрузить демоданные

## 6) Быстрый запуск

```powershell
git clone <repo-url>
cd <repo-folder>
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python -m uvicorn app.main:app --reload
```

Открыть:
- `http://127.0.0.1:8000`

Проверка:
- `GET http://127.0.0.1:8000/health`

## 7) Smoke-проверка

```powershell
python scripts/smoke_scenarios.py
```

Скрипт проверяет ключевые MVP-сценарии: demo, discover/select/analyze, reset состояния, compare channels.

## 8) Почему это готово для хакатона

- Есть полный пользовательский путь от данных до рекомендаций.
- Архитектура простая и расширяемая.
- Безопасность на уровне MVP уже учтена.
- UI и API дают понятную демонстрацию бизнес-ценности для маркетолога.
