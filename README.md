# Content Radar MVP

Content Radar MVP — это веб-инструмент для анализа контента конкурентов, подготовленный в формате MVP для хакатона. Он помогает маркетологам быстро собирать посты, рассчитывать вовлечённость, выявлять наиболее эффективные темы, форматы и время публикаций, а также получать практические рекомендации

## Обзор

Проект реализует простой полный цикл анализа контента:

1. Загрузка постов конкурентов из файла (CSV/JSON) или по ссылке на источник  
2. Парсинг и приведение постов к единой структуре данных  
3. Расчёт метрик вовлечённости  
4. Визуализация результатов в аналитическом дашборде (карточки сводки, таблицы, график, топ-списки)  
5. Формирование кратких рекомендаций для маркетолога

## Какую проблему решает проект

Маркетологи часто анализируют контент конкурентов вручную на разных платформах  
Это занимает много времени и приводит к непоследовательным результатам

Content Radar MVP ускоряет этот процесс, объединяя сбор данных, расчёт метрик, выявление закономерностей и формирование рекомендаций в одном лёгком интерфейсе

## Основные возможности

### Два режима загрузки данных

**1. Загрузка из файла**

Посты можно загрузить через **CSV или JSON**

**2. Получение постов по ссылке**

Сервис может получать посты по **публичной ссылке**  
Если платформа ограничивает доступ к данным, используется **демонстрационный режим (demo fallback)** с явным уведомлением

### Дополнительные функции

- **Нормализация ссылок**  
  Поддерживаются ссылки с протоколом и без него

- **Безопасный слой получения данных**  
  Встроена защита от SSRF-атак и небезопасных локальных адресов

- **Выбор постов перед анализом**  
  Пользователь может выбрать, какие посты использовать в аналитике

### Аналитика вовлечённости

Сервис автоматически определяет:

- лучшие посты  
- наиболее эффективные темы  
- лучшее время публикации  
- наиболее успешные форматы контента

### Визуализация

Результаты отображаются в интерфейсе в виде:

- карточек со сводной информацией  
- таблицы постов  
- графика вовлечённости  
- списка популярных тем  
- рекомендаций

### Ориентация на маркетологов

На выходе сервис формирует **короткие практические рекомендации**, которые можно использовать при планировании контент-стратегии

## Поддерживаемые источники данных

### 1. Загрузка файла

Поддерживаются **JSON и CSV** со следующими полями:

- `text`  
- `likes`  
- `comments`  
- `shares`  
- `views`  
- `date`  
- `format`  
- `competitor` (необязательно)

### 2. Получение данных по ссылке

Поддерживаются:

**Telegram**

Публичные каналы Telegram (`t.me/...`, автоматически нормализуются для веб-просмотра)

**Instagram и другие страницы**

Если на странице доступны открытые метаданные, сервис может извлекать информацию

Если платформа ограничивает доступ к данным, приложение использует **demo fallback** и отображает предупреждение

## Формула вовлечённости

В проекте используется взвешенная метрика:

```
Engagement Score = лайки × 1 + комментарии × 2 + репосты × 3 + просмотры × 0.1
```

Формула также возвращается через API (`engagement_formula`), чтобы фронтенд мог отображать её пользователю

## Текущие ограничения MVP

- Получение данных по ссылке зависит от доступности публичной HTML-разметки страницы  
- Instagram часто ограничивает доступ к метрикам без официального API или авторизации  
- Публичные страницы Telegram могут скрывать некоторые счётчики (например реакции или комментарии)  
- Данные хранятся **в памяти приложения** (in-memory storage), поэтому после перезапуска состояние сбрасывается  
- Определение тем выполняется простым эвристическим методом (TF-IDF-подобный подход) для быстрой работы MVP

## Установка

### 1. Клонирование репозитория

```bash
git clone <your-repo-url>
cd <your-repo-folder>
```

### 2. Создание виртуального окружения

**Windows**

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

**macOS / Linux**

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

## Запуск приложения

```bash
python -m uvicorn app.main:app --reload
```

Открыть в браузере:

```
http://127.0.0.1:8000
```

## Проверка работоспособности

```
GET http://127.0.0.1:8000/health
```

## Демонстрационный сценарий использования

1. Открыть дашборд  
2. Выбрать режим:
   - загрузка файла (CSV/JSON)
   - получение постов по ссылке
3. В режиме ссылки:
   - нажать **Получить посты**
   - выбрать нужные посты
   - нажать **Сохранить выбранные посты**
4. Нажать **Запустить анализ**
5. Просмотреть результаты:
   - карточки сводки
   - график вовлечённости
   - топ постов
   - топ тем
   - лучшие форматы
   - лучшее время публикации
   - рекомендации

## API эндпоинты

- `GET /` — интерфейс дашборда  
- `GET /health` — проверка состояния сервиса  
- `POST /upload_posts` — загрузка CSV/JSON  
- `POST /discover_posts` — получение постов по ссылке  
- `POST /select_posts` — сохранение выбранных постов  
- `POST /analyze_content` — запуск анализа  
- `GET /insights` — рекомендации маркетологу  
- `POST /load_demo` — загрузка демоданных

## Примечания для хакатона

- MVP специально сделан лёгким и удобным для демонстрации  
- Сервис явно сообщает пользователю, если используется demo fallback из-за ограничений платформы  
- Архитектура может быть расширена добавлением базы данных и официальных API платформ
